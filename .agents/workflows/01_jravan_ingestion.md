# JRA-VANデータ取得スキル (Data Ingestion Skill)

このスキルは、穴狙い競馬予想システムにおける「データ取得モジュール」の構築ルールを定義します。

## 目的
JRA-VAN DataLab. を公式・主データソースとして利用し、過去データおよび当日の出馬表・オッズ・天候馬場情報を安定的に取得する。

## 取得対象データ
1. **過去レース結果**: 対象期間10年分以上
    - テーブル: `races`, `results`, `entries` のベースとなるデータ
2. **出馬表**: レース当日の出走馬情報、騎手、斤量等
3. **オッズ**: レース当日の単勝、ワイド、三連複、三連単等のオッズ推移
4. **馬場・天候情報**: 当日の最新の馬場状態

## 設計ルール
- **主軸はJRA-VAN**: データの確実性と拡張性を担保するため、netkeibaへの全面依存は避ける（通信制限・規約リスク回避）。
- **差分更新**: 過去10年分の初期取得後は、週末のみ新しい開催データを差分で取得・DBへ書き込む処理（`ingest_raceday.py`）を構築する。
- **エラーハンドリング**: 通信エラー発生時は一定回数の再試行（リトライ）を行う。
- **netkeibaの扱い**: netkeibaはあくまで「人間が見る用」「血統の補足確認用」として最小限のスクレイピングに留める。
